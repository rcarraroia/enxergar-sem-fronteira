Documento Técnico: Plataforma de Gestão, Atendimento e Acompanhamento
1. Visão Geral do Projeto
O objetivo deste documento é detalhar a arquitetura e as especificações técnicas da plataforma para o Projeto Visão Itinerante. A solução é uma plataforma web para gestão de atendimentos, captação de recursos e acompanhamento de pacientes, com integrações externas e automações.

2. Requisitos e Público-Alvo
Público: Pacientes, Organizadores Locais, Equipe de Gestão (Instituto Coração Valente, Projeto Visão Itinerante, Renum).

Funcionalidades:

Público: Site informativo, agenda de eventos, formulário de cadastro com validação de CPF e consentimento (LGPD), notificações por e-mail/WhatsApp.

Administrativo: Painel de gestão para eventos, relatórios em PDF, gestão de chaves de API do Asaas para Organizadores Locais.

Integrações: API com o Instituto Coração Valente, Asaas (split de 25% por entidade), API de mensagens via WhatsApp.

3. Arquitetura do Sistema
A plataforma será construída com uma arquitetura moderna e escalável, utilizando as seguintes tecnologias:

Frontend: React com TypeScript para uma interface de usuário dinâmica e responsiva.

Backend & Database: Supabase, que oferece funcionalidades de banco de dados (PostgreSQL), autenticação, Edge Functions e armazenamento.

Integrações: API REST para comunicação com o Instituto Coração Valente e o Asaas.

Monitoramento: Painel de Monitoring para acompanhar o status das integrações e o desempenho do sistema.

4. Estrutura de Dados (Modelos de Banco de Dados)
events: Gerenciamento dos eventos de atendimento (local, data, vagas, organizador_id).

patients: Armazena dados dos pacientes (nome, cpf, email, telefone, consentimento_lgpd, diagnostico).

registrations: Tabela de associação entre patients e events.

organizers: Gerencia os organizadores locais e suas chaves de API (asaas_api_key).

asaas_transactions: Logs de transações e split de pagamentos.

instituto_integration_queue: Fila de tarefas para reprocessamento de envios à API do Instituto.

Verificação de vagas disponíveis antes do cadastro.

Painel Administrativo:

CRUD de eventos.

Endpoint GET /api/events/{id}/registrations para lista de inscritos.

Geração de PDF/CSV via biblioteca de terceiros no backend.

Formulário para o organizer inserir e criptografar a chave asaas_api_key.

Módulo de Follow-up (AI Renum):

6. Integrações Externas

Utilização do service de integração já desenvolvido.

Payload de envio com origem_cadastro: "visao_itinerante".

Sistema de retry com Edge Functions e fila dedicada.

API Asaas:

API de pagamentos para processar doações com split dinâmico.

API de WhatsApp:

API externa para envio de mensagens automáticas de confirmação e follow-up.
